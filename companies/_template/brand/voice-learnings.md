# Voice Learnings Log: {COMPANY_NAME}

This file is the self-improving feedback loop for brand voice. After every content cycle, append learnings here. Claude reads this file during content generation to avoid repeating mistakes and double down on what works.

**Authority level**: This file can override default voice/format preferences in `voice.md` based on performance data, but it can NEVER override compliance rules.

---

## When to Read This File

- **BEFORE generating any content** for any account (main or satellite). This is mandatory.
- **BEFORE weekly voice calibration**
- **BEFORE monthly voice recalibration**
- **After any crisis** to review communication learnings

## How to Apply Learnings

1. **Individual entries**: Check the most recent entries for your target account. Apply any REPEAT actions. Avoid anything tagged AVOID. Consider TEST AGAIN items.
2. **Weekly summaries**: Use the latest weekly summary to inform content planning. Prioritize formats and hooks that performed well.
3. **Monthly calibrations**: Check if any voice weight changes have been recommended.
4. **A/B test results**: Check active tests before generating content.
5. **Competitor watch**: Review the latest competitor observations for positioning adjustments.
